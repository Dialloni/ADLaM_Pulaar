01 Quick wins — do this week

P0
Brand consistency · 2 hrs

Replace the Sparkles icon — it reads as Gemini.

Sparkles from lucide-react appears 10 times across App.tsx and Chat.tsx — login hero, empty states, the "Generate" CTA, avatar for bot messages, and the beta chip. It's the exact four-point mark Google uses for Gemini, so the product reads as a Gemini wrapper.

App.tsx:315, 472, 739, 908, 1004, 1018 Chat.tsx:293, 649
Fix Swap every <Sparkles/> for the new Ga Hook mark. Commit an <GandoMark/> component and import it everywhere the sparkle was.

P0
Tokens · 3 hrs

Stop hard-coding the brand hex — use CSS variables.

#ff8b9b and #fd8b00 appear ~30 times inline in components, defeating the @theme tokens in index.css. When the brand changes (or when dark/light variants are added), every occurrence must be edited. The top of App.tsx even redeclares const P = '#ff8b9b'; const S = '#fd8b00'; as module constants.

App.tsx:30-31 + 5 refs Chat.tsx: 20+ occurrences LanguageSelector.tsx:80
Fix Expose the theme variables to Tailwind utilities and replace inline hex with text-primary / bg-primary. Tailwind v4's @theme already does this — you're just not using the generated classes. 
// before
className="text-[#ff8b9b] border-[#ff8b9b]/20"

// after
className="text-primary border-primary/20"
P0
A11y — critical · 4 hrs

Icon buttons have no accessible name. Screen readers announce nothing.

Header bell, profile toggle, delete-project, revert, copy, and the send button are all <button>s wrapping a lucide icon — no aria-label, no text. A Fulani speaker on a screen reader hears "button, button, button."

App.tsx: header icon cluster, project-card overlay actions Chat.tsx: send, mic, copy, revert, thumbs up/down
Fix Add aria-label={t.xxx} to every icon-only button. Run axe-core once (npm i -D @axe-core/react, one line in main.tsx) to catch the rest. Also: the img alt="" on user photos is correct (decorative) — keep that — but the Google favicon has alt="Google" which needs translation.
P0
A11y — contrast · 1 hr

Body text #adaaaa on #0e0e0e passes AA — but #767575, #52525b, and #3f3f46 don't.

Your eyebrow labels, nav inactive state, meta captions, and footer text sit at 2.8:1 – 3.9:1 contrast — below WCAG AA (4.5:1) for body copy. Only #adaaaa (7.1:1) and #fff are safe for text ≥ 12px.

Fix Bump the low-contrast grays up one step for text, keep the existing values for strokes/dividers. A single find-and-replace in index.css @theme: 
--color-fg-3: #8b8a8a; /* was #767575, nav inactive */
--color-fg-4: #6b6b70; /* was #52525b, meta copy   */

02 Architecture — needs a refactor sprint

P1
Structure · 2 days

Split App.tsx (1,046 lines) into routed pages.

App.tsx holds auth, dashboard, projects list, workspace, status, docs, header, sidebar, and 7 state-machine useEffects in a single file. There's a manual const [page, setPage] = useState<NavPage>('dashboard') instead of React Router. You're already at 19 useState hooks on the root — it's unmaintainable.

Fix — target shape 
src/
 ├ routes/
 │  ├ Login.tsx        // auth UI
 │  ├ Dashboard.tsx    // stats, recents
 │  ├ Projects.tsx     // grid + search
 │  ├ Workspace.tsx    // chat + preview
 │  ├ Status.tsx
 │  └ Docs.tsx
 ├ layout/
 │  ├ AppShell.tsx     // header + sidebar + <Outlet/>
 │  ├ Header.tsx
 │  └ Sidebar.tsx
 ├ components/ // (as today)
 └ stores/
    ├ useProjects.ts   // zustand or useReducer
    └ useStatus.ts
Why Each route becomes ~150 lines. react-router-dom gets you browser-history, deep-linking to a project, and back-button support (all broken today).
P1
Data · 1 day

The Firestore listener re-subscribes on every currentProject.id change — and the stale-check inside does double work.

App.tsx:145-155 — the projects onSnapshot effect has [user, currentProject?.id] as deps. Every time the user opens a project, you tear down and re-subscribe to the entire projects collection. It also runs a list.find(p => p.id === currentProject.id) on every snapshot to patch currentProject — unnecessary because you can read it from the map directly.

Fix Two effects: one subscribing to projects with deps [user?.uid] only; a second derived useMemo that reads currentProject from the list by id. Extract to a useProjects() hook.
P1
Error UX · 4 hrs

Errors are logged to the console, not shown to the user.

console.error is called in 11 places — rename failure, delete failure, code-update failure, mic failure, transcription failure. The user sees nothing. For a bilingual product this is worse: the user may assume the click was received when it wasn't.

App.tsx:246, 264, 270 Chat.tsx:231, 244 AuthContext.tsx:37,54,63,72,81
Fix Add a toast system (sonner or a 50-line own one) and wire setGlobalError to it. Show a localized string from translations.ts, not the raw Firestore error.
P1
Input UX · 1 hr

confirm(t.confirmDelete) is a native browser dialog — not brand, not translated consistently, not mobile-friendly.

App.tsx:269. Native confirm() styling ignores the brand, sits above glass panels ugly, and some mobile browsers localize the OK/Cancel buttons separately from your t string.

Fix Ship a <ConfirmDialog/> modal using your existing .glass + rounded-3xl vocabulary. Keyboard-accessible (Esc closes, Enter confirms, focus trap).

03 Perf — stability & bundle

P2
Bundle · 2 hrs

All 22 lucide icons are tree-shaken per-import already — but motion, prismjs, and react-simple-code-editor load on first paint.

The workspace (CodeEditor, chat markdown with react-markdown) is only reached after login + opening a project, yet its dependencies sit in the initial bundle. Lighthouse's TBT will thank you.

Fix React.lazy the CodeEditor and Preview iframe. Estimated shave: 180–220 kB gzipped.
P2
Re-renders · 2 hrs

No useMemo on filtered/derived arrays — they recompute on every render.

filteredProjects, userMessages, completionPct, tokenPct are all computed inline in the render body. Paired with the 19 useStates, App re-renders many times per second while the user types in the chat.

Fix Wrap in useMemo(..., [projects, projectSearch]). Bigger win comes from the route-split above.
P2
Layout · 3 hrs

Workspace split-pane isn't responsive — below ~900 px, chat and preview stack awkwardly without a tab switcher.

The split is a grid-cols-1 lg:grid-cols-2. Below lg, both panes render stacked — on a 390×844 phone that's 2,000 px of scroll before the preview is reachable.

Fix Add a mobile tab switcher (Chat / Preview / Code) and hide the inactive panes with display:none — not opacity. Reuse the existing activeTab state.

04 Polish — when the roadmap allows

P3
i18n hygiene · 2 hrs

Hardcoded English in at least 8 places — "Beta", "Operational", "Degraded", "Projects", error strings.

You've reached the AI limit. Please wait a minute. is English-only even when ADLaM is selected. Same for the system-status labels and the sidebar "Workspace" chip.

Fix Thread the missing strings through TRANSLATIONS. Add a CI check: any string literal over 3 chars in JSX must come from t.*.
P3
Motion · 2 hrs

Two animation libraries in play (motion/react + CSS keyframes).

Spring config is duplicated. Easing is hard-coded [0.16, 1, 0.3, 1] in 9 places.

Fix Export a src/motion.ts with EASE_STANDARD, SPRING_CARD, FADE_IN. Same as CSS vars, but for Framer.
P3
Empty states · half-day

Empty dashboard (0 projects) and empty search both show the same block of chrome — no guidance.

First-run users see stat chips reading 0 with no nudge toward the chat input.

Fix Different empty state per surface. Dashboard: giant CTA "Describe your first app →" that focuses the chat. Projects: starter template cards. Status: "Checking…" Lottie (already there, keep).
P3
Security · 1 hr

The generated .html is previewed in an iframe without sandbox.

If Gemini returns code with an inline <script> that calls fetch to a third-party, it runs with your origin's cookies. Not exploited yet because Firebase auth is HTTP-only, but a drive-by of localStorage is trivial.

Fix <iframe sandbox="allow-scripts" srcDoc={code}/>. If you want forms/popups later, widen from there — don't open-default.
